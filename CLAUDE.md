# CLAUDE.md — bs (BankSalad Custom Budget Tracker)

> 이 파일은 Claude Code가 프로젝트 작업 시 항상 참조하는 컨텍스트 문서다.
> 마지막 업데이트: 2026-03-07

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **저장소** | `bs` |
| **목적** | 뱅크샐러드 앱의 데이터를 import하여, 뱅크샐러드가 제공하지 않는 **대분류별 세분화 예산 관리**와 **싱킹펀드 추적**을 수행하는 개인 재무 대시보드 |
| **핵심 워크플로우** | 뱅크샐러드(데이터 수집) → 엑셀 내보내기 → 이 앱에 import → 예산 대비 분석 |
| **사용자** | 이명준 (평창, 산림청 공무원) |
| **디자인 레퍼런스** | 뱅크샐러드 앱 UI (스크린샷 65장 보유, 단계별 참조 예정) |

| 항목 | 선택 |

## 뱅크샐러드 엑셀 데이터 포맷

### 시트 1: `뱅샐현황`

월별 수입/지출 요약, 자산/부채, 보험, 투자, 대출 현황 포함.
- Row 11: 월 헤더 (2025-03 ~ 2026-03 형태, 13개월)
- Row 12~19: 수입 카테고리 (금융수입, 급여, 기타수입, 더치페이, 사업수입, 상여금, 용돈, 미분류)
- Row 20: 월수입 총계
- Row 21~37: 지출 카테고리 (17개 대분류)
- Row 38: 월지출 총계
- Row 39: 순수입 총계
- Row 44~82: 자산/부채 현황
- Row 85~: 보험, 투자, 대출 현황

### 시트 2: `가계부 내역`

개별 거래 기록 (1,600건+).

| 컬럼 | 타입 | 예시 |
|------|------|------|
| 날짜 | datetime | 2026-03-07 |
| 시간 | time | 16:10:17 |
| 타입 | string | 지출 / 수입 |
| 대분류 | string | 식비 |
| 소분류 | string | 식재료 |
| 내용 | string | 하나로마트사업소 |
| 금액 | number | -16210 (지출은 음수) |
| 화폐 | string | KRW |
| 결제수단 | string | 올바른NEW HAVE체크 |
| 메모 | string | (nullable) |

## 싱킹펀드 (Sinking Fund) 추적

| 항목 | 연간 목표 | 월 적립액 | 납부 시기 | 비고 |
|------|----------|----------|----------|------|
| 자동차 보험 | 700,000원 | 58,333원 | 9월 | 삼성화재 애니카 |
| 차량 정비 | 600,000원 | 50,000원 | 비정기 | 예비비 성격 |

### 싱킹펀드 UI 요구사항
- 각 항목별 프로그레스 바 (적립액/목표액)
- 적립액 수동 입력 또는 자동 계산 (월 적립액 × 경과 월수)
- 납부 완료 시 리셋 기능
- 납부 시기 D-day 표시

## 앱 화면 구조 (예상)

### 1. 메인 대시보드
- 이번 달 총 예산 vs 총 지출 (원형 또는 바 차트)
- 대분류별 예산 소진율 카드 리스트 (프로그레스 바)
- 잔여 예산 / 초과 알림
- 최근 거래 미리보기

### 2. 카테고리 상세
- 특정 대분류 클릭 시 → 소분류별 지출 내역
- 해당 카테고리 예산 대비 추이 차트
- 거래 목록 (날짜, 내용, 금액, 결제수단)

### 3. 싱킹펀드 대시보드
- 4개 항목 프로그레스 카드
- 적립 현황 / 목표 / D-day
- 납부 기록

### 4. 예산 설정
- 17개 대분류별 월 예산 입력/수정
- 싱킹펀드 항목 관리

### 5. 데이터 Import
- 뱅크샐러드 엑셀 파일 업로드 (드래그앤드롭)
- 파싱 미리보기 → 확인 후 저장
- 기존 데이터와 병합 또는 교체 선택

### 6. 월별/연간 리포트
- 월 선택 네비게이션
- 수입/지출 추이
- 카테고리별 비중 변화

## 디자인 방향

- **레퍼런스:** 뱅크샐러드 앱 (스크린샷 65장 단계별 참조)
- **테마:** 밝은 배경(화이트/라이트그레이) + 컬러 포인트 (뱅크샐러드 스타일)
- **레이아웃:** 카드 기반, 둥근 모서리, 적절한 그림자
- **타이포그래피:** Pretendard 또는 Noto Sans KR
- **모바일 우선:** max-width 제한, 터치 친화적
- **차트:** 부드러운 곡선, 뱅크샐러드 특유의 그라데이션 컬러

## 코딩 규칙

- 컴포넌트는 함수형, Hooks 사용
- TypeScript strict mode
- 파일 구조: feature 기반 (pages/, components/, hooks/, utils/, types/)
- 한국어 주석 사용
- 데이터 파싱 로직은 utils/로 분리
- 상태 관리: React Context 또는 Zustand (규모에 따라 결정)

## 개발 우선순위

1. **프로젝트 초기화** — Vite + React + TS + Tailwind 세팅
2. **엑셀 파싱** — SheetJS로 뱅크샐러드 엑셀 import 로직
3. **예산 설정 화면** — 17개 대분류별 예산 CRUD
4. **메인 대시보드** — 예산 vs 실지출 비교 뷰
5. **싱킹펀드 대시보드** — 4개 항목 적립 추적
6. **카테고리 상세** — 소분류별 거래 내역
7. **월별 리포트** — 추이 차트
8. **디자인 폴리싱** — 뱅크샐러드 스크린샷 참조하여 UI 리파인
